What is a Target Group?

A Target Group is a logical group of backend resources that receive traffic from a Load Balancer.

The Load Balancer sends incoming requests to the healthy targets in the Target Group.


Target Types

A Target Group can contain:

EC2 Instances
IP Addresses
Lambda Functions
Application Load Balancers (for specific use cases)

Why Target Groups?

Instead of sending traffic directly to EC2 instances, the Load Balancer sends traffic to a Target Group.

Benefits:

Health checks
Easy scaling
Add/remove servers without changing the Load Balancer
High availability

Useful for hosting multiple applications behind one Load Balancer.


What is a Network Load Balancer (NLB)?

A Network Load Balancer (NLB) distributes traffic at Layer 4 (Transport Layer) using:

TCP
UDP
TLS

It is designed for very high performance and low latency.


NLB Architecture

Client

↓

Network Load Balancer

↓

Target Group

↓

EC2 Instances


Features of NLB
Layer 4 Load Balancer
Supports TCP, UDP, TLS
Very high performance
Handles millions of requests
Low latency
Static IP support
Preserves client IP
Multi-AZ support

NLB vs ALB

| Network Load Balancer          | Application Load Balancer            |
| ------------------------------ | ------------------------------------ |
| Layer 4                        | Layer 7                              |
| TCP, UDP, TLS                  | HTTP, HTTPS                          |
| Extremely fast                 | Content-aware routing                |
| Static IP                      | No static IP (by default)            |
| Preserves client IP            | Supports path and host-based routing |
| Best for high-performance apps | Best for web applications            |
