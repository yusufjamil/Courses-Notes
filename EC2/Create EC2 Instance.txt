Step 6: Create a Security Group

A Security Group acts as a virtual firewall for the EC2 instance.

Add the following inbound rules:

| Type  | Port | Source    | Purpose            |
| ----- | ---- | --------- | ------------------ |
| SSH   | 22   | My IP     | Remote login       |
| HTTP  | 80   | 0.0.0.0/0 | Web server access  |
| HTTPS | 443  | 0.0.0.0/0 | Secure web traffic |

Rule Explanations

SSH (22):
Allows secure remote administration.

HTTP (80):
Allows web traffic over HTTP.

HTTPS (443):
Allows encrypted web traffic.

Important: Without an SSH rule, EC2 Instance Connect and SSH clients cannot connect to the instance.

Understanding sudo

It allows an authorized user to execute commands with root privileges.

Always exercise caution when operating as the root user, as commands executed with root privileges can affect the entire system.

Common Beginner Mistakes
Forgetting to allow SSH (Port 22) in the Security Group.
Not assigning a public IP, preventing internet access.
Using the wrong username (for Amazon Linux, the default is ec2-user).
Running administrative commands without sudo, leading to permission errors.
Leaving instances running after completing a lab, resulting in additional charges.
Deleting an instance without realizing that termination is irreversible.

Interview Questions

1. What is a Key Pair in AWS?

A Key Pair consists of a public key stored by AWS and a private key downloaded by the user. It is used for secure authentication to Linux EC2 instances instead of passwords.

2. What is the difference between RSA and ED25519?

| RSA                     | ED25519                    |
| ----------------------- | -------------------------- |
| Older, widely supported | Newer, faster, more secure |
| Larger key size         | Smaller key size           |
| Broad compatibility     | Modern SSH implementations |


3. What is the difference between .pem and .ppk files?
.pem: Used with OpenSSH on Linux, macOS, and many SSH clients.
.ppk: Used with PuTTY on Windows.

4. Why should SSH not be open to 0.0.0.0/0?

Opening SSH to 0.0.0.0/0 allows connection attempts from any IP address on the internet, increasing the attack surface. Restricting SSH to My IP or trusted networks is more secure.

5. What is the purpose of a Security Group?

A Security Group acts as a stateful virtual firewall for an EC2 instance, controlling allowed inbound and outbound network traffic.

6. What is the difference between a Public IP and a Private IP?
A Public IP is internet-routable and used for external access.
A Private IP is only reachable within the VPC and is used for internal communication between AWS resources.

Termination Protection Error

If AWS displays an error similar to:

Failed to enable termination protection...

Possible reasons include:

The instance is already terminating.
The instance is in an invalid lifecycle state.
The operation was attempted after termination had begun.

Termination protection can only be changed when the instance is in a supported state (such as Running or Stopped), not while it is already being deleted.

Interview Questions

1. What is Amazon EBS?

Amazon Elastic Block Store (EBS) provides persistent block storage for EC2 instances. Data remains available even after the instance is stopped.

2. What is the difference between Stop and Terminate?

| Stop                      | Terminate                             |
| ------------------------- | ------------------------------------- |
| Instance can be restarted | Instance is permanently deleted       |
| Root EBS volume remains   | Root EBS volume is deleted by default |
| Compute charges stop      | All compute resources are removed     |

3. What is gp3?

gp3 is AWS's recommended general-purpose SSD EBS volume type, offering consistent performance, configurable IOPS, and lower cost than the older gp2 for many workloads.

4. What is Termination Protection?

Termination Protection is an EC2 feature that prevents accidental deletion of an instance. It must be disabled before the instance can be terminated.

5. Does stopping an EC2 instance delete data?

No. Stopping an instance does not delete the attached EBS volumes. The operating system and data remain intact unless the instance is terminated and the volume is configured to be deleted.

6. What happens to the public IP when an instance is stopped?

If the instance uses an automatically assigned public IPv4 address, it is typically released when the instance stops and a new public IP is assigned when it starts again. An Elastic IP remains associated until you release or reassign it.
