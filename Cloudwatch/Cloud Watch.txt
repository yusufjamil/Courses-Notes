What is Amazon CloudWatch?

Amazon CloudWatch is a monitoring and observability service that collects metrics, logs, and events from AWS resources. It helps monitor resource health, trigger alarms, automate actions, and notify administrators.

Workflow
EC2 publishes metrics (CPU, Network, Disk).
CloudWatch collects those metrics.
CloudWatch compares them with the configured threshold.
If the threshold is exceeded, the alarm changes to ALARM state.
CloudWatch sends a notification to an SNS Topic.
SNS delivers notifications to subscribers.

Threshold Type

90

Meaning

If CPU utilization exceeds 90%, the alarm will trigger.

Threshold Options

CloudWatch supports:

| Option        | Meaning            |
| ------------- | ------------------ |
| Greater       | Metric > Threshold |
| Greater/Equal | Metric ≥ Threshold |
| Lower         | Metric < Threshold |
| Lower/Equal   | Metric ≤ Threshold |

Anomaly Detection

Instead of a fixed threshold you can choose

Anomaly Detection

CloudWatch learns normal behavior and automatically detects unusual spikes.

Example:

Instead of:

CPU > 90%

CloudWatch detects abnormal CPU behavior automatically.

Additional Alarm Actions

Besides notifications, CloudWatch can automatically:

Stop an EC2 instance
Reboot an EC2 instance
Terminate an EC2 instance
Trigger an Auto Scaling policy
Invoke a Lambda function
Create Systems Manager OpsItems
