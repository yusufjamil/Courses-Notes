What is Serverless?

Serverless is a cloud computing model where AWS manages the underlying servers, operating systems, scaling, and infrastructure. Developers only focus on writing and deploying code.

Note: "Serverless" does not mean there are no servers. It means you don't manage the servers—AWS does.

What is AWS Lambda?

AWS Lambda is a serverless compute service that lets you run code without provisioning or managing servers.

AWS automatically:

Provisions servers
Scales applications
Monitors execution
Handles availability
Manages infrastructure

You only upload your code.

Lambda Components

The code you write.

Example:

    return {
        "statusCode":200,
        "body":"Hello World"
    }

Event

The input sent to Lambda.

Example


{
   "name":"Yousuf"
}


Context

Provides runtime information such as:

Function name
Remaining execution time
Request ID
Memory allocated

Lambda Pricing

You pay only for:

Number of requests
Execution duration (milliseconds)
Memory allocated

No charges when the function is idle.


Stateless Nature

Each Lambda execution is independent.

Example

Request 1

↓

Lambda

↓

Ends

Request 2

↓

New Execution

Data is not stored between executions unless you use services like DynamoDB or S3.


Scaling in Lambda

Lambda automatically creates multiple instances of the function.


100 Requests

↓

100 Lambda Executions

No manual scaling required.

Lambda vs EC2

| AWS Lambda          | Amazon EC2                      |
| ------------------- | ------------------------------- |
| Serverless          | Virtual Server                  |
| AWS manages servers | User manages servers            |
| Automatic scaling   | Manual or Auto Scaling          |
| Pay per execution   | Pay while instance runs         |
| Event-driven        | Always running (unless stopped) |
| No OS management    | OS management required          |


Serverless vs Traditional Servers


| Serverless           | Traditional Server     |
| -------------------- | ---------------------- |
| No server management | User manages servers   |
| Automatic scaling    | Manual scaling         |
| Pay per execution    | Pay for running server |
| Faster deployment    | Longer setup           |
| Event-driven         | Server-based           |
