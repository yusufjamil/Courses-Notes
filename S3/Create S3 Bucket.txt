What is an S3 Bucket?

An Amazon S3 Bucket is a logical container used to store objects (files) in Amazon S3.

Every file uploaded to Amazon S3 must reside inside a bucket.

Examples of objects stored in a bucket include:

Images
Videos
PDF documents
ZIP archives
Application files
Log files
Backup files
Step 1 – Open Amazon S3

After logging in to the AWS Management Console:

Use the Search bar.
Search for S3.
Open the Amazon S3 service.

The S3 dashboard displays:

Existing buckets
Storage overview
Access points
Recent activity

If this is a new account, no buckets will be listed.

Step 2 – Create Bucket

Click the Create bucket button.

AWS opens the bucket creation wizard.

This wizard guides you through all required configuration options before the bucket is created.

Step 3 – Choose AWS Region

Every bucket is created in an AWS Region.

Examples:

Asia Pacific (Mumbai)
Asia Pacific (Singapore)
US East (N. Virginia)
Europe (Frankfurt)
Why the Region Matters

The selected Region determines:

Where your data is stored.
Data residency and compliance.
Latency for users.
Potential data transfer costs.

Best Practice: Choose the Region closest to your users or where your application runs.

Step 4 – Bucket Name

The bucket name must be entered during creation.

Bucket Naming Rules
Must be globally unique across AWS.
Use only lowercase letters.
Numbers are allowed.
Hyphens (-) are allowed.
No spaces.
No uppercase letters.
No special characters other than hyphens (and periods in some cases, though they are generally avoided).
Examples

Valid

company-backups
student-project-files
aws-training-demo

Invalid

MyBucket
AWS Bucket
Backup_Files

Step 5 – Object Ownership

The wizard includes an Object Ownership section.

This setting determines who owns newly uploaded objects.

The commonly recommended option is Bucket owner enforced, which:

Makes the bucket owner the owner of all uploaded objects.
Disables Access Control Lists (ACLs).
Simplifies permission management.

For most new projects and AWS best practices, this is the preferred configuration.

Step 6 – Block Public Access

The video shows the Block Public Access configuration.

This is one of the most important security settings in Amazon S3.

Purpose

It helps prevent accidental exposure of bucket contents to the internet.

When enabled:

Public bucket policies are blocked.
Public ACLs are ignored.
Anonymous users cannot access bucket contents.

For most applications, these settings should remain enabled.

Only disable them when you intentionally need a public bucket (for example, certain static website hosting scenarios).

Step 7 – Bucket Versioning

The wizard also includes Bucket Versioning.

What is Versioning?

Versioning allows Amazon S3 to keep multiple versions of the same object.

Example

Upload:

report.pdf

Later, upload another file with the same name.

Without versioning:

The original file is overwritten.

With versioning:

Both versions are preserved.
Older versions can be restored if needed.

Benefits
Protects against accidental deletion.
Protects against accidental overwrite.
Simplifies recovery of previous file versions.

For production environments, versioning is commonly enabled.

Step 8 – Review Configuration

Before creating the bucket, review:

Bucket name
Selected Region
Object Ownership
Public access settings
Versioning

If everything is correct, proceed.

Step 9 – Create Bucket

Click Create bucket.

If all validation checks pass, AWS creates the bucket and it appears in the S3 bucket list.

The bucket is now ready to store objects.

Bucket Lifecycle After Creation

Once created, you can:

Upload files.
Create folders (prefixes).
Download objects.
Delete objects.
Configure permissions.
Enable encryption.
Apply lifecycle rules.
Configure replication.
Host static websites (if required).

Common Beginner Mistakes
Choosing a bucket name that is already in use.
Selecting the wrong AWS Region.
Disabling Block Public Access unnecessarily.
Forgetting to enable Versioning for important data.
Using unclear bucket names that make management difficult.

Best Practices
Use descriptive, globally unique bucket names.
Keep Block Public Access enabled unless public access is explicitly required.
Enable Versioning for important or production data.
Use IAM policies instead of broad public permissions.
Enable server-side encryption for sensitive data.
Organize objects with logical prefixes (for example, images/, backups/, logs/).

Real-World Example

A company stores customer invoices in an S3 bucket.

Bucket name: company-invoices
Region: Asia Pacific (Mumbai)
Versioning: Enabled
Block Public Access: Enabled

If an employee accidentally uploads an incorrect invoice over an existing one, versioning allows the previous version to be restored.

Interview Questions

1. What is an S3 bucket?

An S3 bucket is a logical container used to store objects in Amazon S3.

2. Why must bucket names be globally unique?

Because bucket names form part of the bucket's unique namespace within Amazon S3.

3. Why should Block Public Access usually remain enabled?

To prevent accidental public exposure of sensitive data and reduce security risks.

4. What is Object Ownership?

It determines who owns objects uploaded to the bucket. The recommended setting is Bucket owner enforced, which simplifies access management by disabling ACLs.

5. What is Bucket Versioning?

A feature that stores multiple versions of an object, helping recover from accidental deletion or overwriting.

6. Can the AWS Region be changed after creating a bucket?

No. A bucket remains in the Region where it was created. To use another Region, you typically create a new bucket there and migrate the data.

Quick Revision

Use Bucket owner enforced for Object Ownership in most cases.
Leave Block Public Access enabled unless public access is intentionally required.
Enable Versioning to protect against accidental overwrites and deletions.

What is an Object?

Everything stored inside an Amazon S3 bucket is called an Object.

Examples of objects include:

PDF files
Images
Videos
ZIP files
Excel sheets
Word documents
Text files
Backup files
Log files

Unlike a traditional file system, Amazon S3 stores data as objects, each containing:

The actual file data
Metadata
A unique object key
Opening an S3 Bucket

Navigate to:

AWS Console → S3 → Buckets

A list of available buckets is displayed.

Click the bucket you created in the previous lesson.

The bucket opens and displays its contents.

If it's a newly created bucket, it will initially be empty.

Uploading Objects

To upload files:

Open the bucket.
Click Upload.
Select Add files (or Add folder, if supported).
Browse your local computer.
Select one or more files.
Click Upload.

AWS transfers the selected files to Amazon S3.

Once complete, the objects appear in the bucket.

Uploading Multiple Files

Amazon S3 allows uploading:

A single file
Multiple files
Entire folders (through supported interfaces or tools)

This is useful when migrating application assets, backups, or datasets.

Object Key

Every uploaded object receives a unique Object Key.

Example:

Bucket:

student-records

Objects:

student-records
├── student1.pdf
├── student2.pdf
├── photo.jpg
└── resume.docx

Here:

student1.pdf
student2.pdf
photo.jpg
resume.docx

are object keys.

Object Details

Clicking an object displays additional information, such as:

Object name
File size
Last modified date
Storage class
Object URL
Owner
Metadata
Encryption status

This information helps with management and troubleshooting.

Object URL

Each object has a URL that can be used to access it.

However, having a URL does not mean the object is publicly accessible.

Access depends on:

IAM permissions
Bucket policy
Object ACLs (if enabled)
Block Public Access settings
Metadata

Metadata is additional information associated with an object.

Examples include:

Content type (e.g., image/jpeg, application/pdf)
Content length
Last modified time
Custom metadata added by applications

Metadata helps applications identify and process files correctly.

Storage Class

Every object belongs to a storage class.

Common storage classes include:

S3 Standard
S3 Intelligent-Tiering
S3 Standard-IA
S3 One Zone-IA
S3 Glacier Instant Retrieval
S3 Glacier Flexible Retrieval
S3 Glacier Deep Archive

By default, uploaded objects use S3 Standard unless another class is selected.

Downloading Objects

To download an object:

Open the bucket.
Select the object.
Click Download.

The file is downloaded to your local machine.

Deleting Objects

To remove a file:

Select the object.
Click Delete.
Confirm the deletion.

The object is removed from the bucket.

If Versioning is enabled, AWS creates a delete marker instead of permanently removing the object immediately.

Creating Folders

Although Amazon S3 is an object storage service and does not use real directories, the console allows users to create folders.

Internally, folders are represented using prefixes.

Example:

company-data
│
├── Images/
│      logo.png
│      banner.jpg
│
├── Documents/
│      invoice.pdf
│      contract.docx
│
└── Backups/
       backup.zip

Folders improve organization and make objects easier to locate.

Interview Questions
1. What is an Object in Amazon S3?

An object is the basic unit of storage in Amazon S3. It consists of the file data, metadata, and a unique object key.

2. What is an Object Key?

The unique identifier (name) of an object within a bucket.

3. Can you upload multiple files to Amazon S3?

Yes. Amazon S3 supports uploading one file, multiple files, and folders (depending on the upload method).

4. Are S3 folders real directories?

No. They are logical prefixes used to organize object keys.

5. What information can you view for an object?
Name
Size
Last modified
Storage class
Metadata
URL
Encryption
Permissions

6. Does having an object URL make it public?

No. Access depends on bucket policies, IAM permissions, Block Public Access settings, and other access controls.

7. What happens when Versioning is enabled and an object is deleted?

Amazon S3 adds a delete marker rather than immediately removing the object's previous versions, allowing recovery if needed.

Cloud Practitioner Exam Tips
Remember that S3 is an object storage service.
Buckets contain objects, not traditional files and folders.
Folders shown in the AWS Console are prefixes, not physical directories.
By default, uploaded objects use the S3 Standard storage class.
Public access is controlled through IAM, bucket policies, ACLs (if used), and Block Public Access settings.
Versioning protects against accidental deletion and overwrites.
